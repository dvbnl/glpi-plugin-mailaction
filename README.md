## What it does

MailAction adds a **MailAction** tab to every ticket in GLPI. From this tab, you can compose and send a professionally formatted email containing ticket details, tasks, and followups to anyone involved in the ticket.

### Features

- **Smart recipient selection** - All ticket requesters, technicians, and observers (both GLPI users and external email addresses) are listed as checkboxes
- **Custom addresses** - Add any external email address as an additional recipient
- **Privacy control** - Option to hide private tasks and followups from the email (enabled by default)
- **Rich text editor** - Full TinyMCE editor for composing the email body
- **Rich ticket data** - Email body is pre-filled with ticket metadata (status, priority, urgency, impact, category, dates, people) plus content, tasks, and followups
- **Customizable HTML template** - Configure a custom HTML email template via Setup > Plugins > MailAction
- **GLPI notification tags** - Use all standard GLPI tags (`##ticket.title##`, `##ticket.status##`, `##FOREACHtasks##`, etc.) in custom templates
- **Email preview** - Preview the fully rendered email (including resolved GLPI tags) before sending
- **Full audit trail** - Every sent email is logged as a ticket task with the email body attached as a document
- **Private tasks by default** - Tasks created by MailAction are private by default, with a toggle in plugin settings
- **Profile-based access** - Control which GLPI profiles can see the MailAction tab
- **Sender selection** - Choose from the GLPI notification sender address, admin address, or your own email

## Installation

1. Download the latest release
2. Extract and rename the folder to `mailaction`
3. Place it in your GLPI `plugins/` directory
4. Go to **Setup > Plugins** and click **Install**, then **Enable**
5. Go to **Administration > Profiles**, select a profile, and enable the **MailAction** tab

### Upgrading

Place the new files over the existing plugin folder and go to **Setup > Plugins** to run any database migrations.

## Configuration

### Email template

Go to **Setup > Plugins**, click the wrench icon next to MailAction to open the template editor.

The template supports two types of placeholders:

| Placeholder | Description |
|-------------|-------------|
| `{{SUBJECT}}` | The email subject line |
| `{{CONTENT}}` | The rich ticket content generated by MailAction |
| `##ticket.title##` | GLPI notification tag (ticket title) |
| `##ticket.status##` | GLPI notification tag (ticket status) |
| `##FOREACHtasks##` ... `##ENDFOREACHtasks##` | Loop over tasks |
| `##IFticket.category##` ... `##ENDIFticket.category##` | Conditional block |

All standard GLPI notification tags for tickets are supported. See GLPI's notification template documentation for the full list.

### Private tasks

By default, tasks created by MailAction (logging that an email was sent) are marked as private. You can toggle this in **Setup > Plugins > MailAction** under "Private tasks".

### Permissions

Go to **Administration > Profiles**, select a profile, and open the **MailAction** tab to enable or disable access per profile.

## Usage

1. Open any ticket in GLPI
2. Click the **MailAction** tab
3. Select a sender address from the dropdown
4. Check the recipients you want to email, or enter a custom address
5. Uncheck "Hide private tasks and followups" if you want to include private content (hidden by default)
6. Edit the subject and message body as needed
7. Click **Preview** to see how the email will look
8. Click **Send**

The email will be sent and a task will be created on the ticket documenting the action.
